Introduction back to ToC

The Timebank Ontology can be used to describe Timebank systems. A Timebank allows user to store virtual money, often called a Time Dollar, into a bank account [Cahn2004]. This virtual currency can be earned by helping fellow Timebank users and spend by requesting help from others.
The two most central concepts are the users and requests. A user can request help and help fellow users. The property isOriginatorOf is used to link a request to a user asking for help. The property accepts is used to link a helper to a request. A user can have skills and limitations, which are linked to the user with the properties hasSkill and hasLimitation. A request can also have skills and limitations associated to it, using the the properties requiresSkill and excludesByLimitation. A request also has an environment, which can be expressed with the property hasEnvironment.
To keep track of the time-balance of a user, each request is worth a specific amount of time. The property hasAccreditedDuration is used to express this amount of time. The current time-balance of a user is calculated by adding all time durations for which the user was a helper (acceppted a request) and substract all time durations for which the user was a receiver of help (originator of a request). The initial time-balance can be expressed by the property hasTimeBalance.
The Timebank Ontology is supported by the Innovation and Technology Fund (ITF) of The Government of the Hong Kong Special Administrative Region under grant Ref: MRP/029/18.
