Introduction back to ToC

The Timebank Ontology is used to capture all relevant context of users and requests of a Timebank system. A Timebank allows user to accumulate and store virtual money, often called a Time Dollar, into a bank account [Cahn2004]. Whenever a user helps a fellow user with some request, a certain amount of this virtual money is transferred from the originator of the request to the helper. The amount of virtual money that is transferred equals exactly the amount of time needed to complete the request.

To decide whether a user is suitable to help one of its fellow user, we need to know:

  • What are the skills of the Timebank users?
  • What are the required skills for a specific request?
  • What are the limitations of the Timebank users?
  • What limitations exclude a user from accepting a request?
  • What are the environmental factors of a request?
  • When is a user available for helping others?

The two most central concepts are the users and requests. A user can request help and offer help to fellow users. The property isOriginatorOf is used to link a request to a user asking for help. The property accepts is used to link a helper to a request. A user can have skills and limitations, which are linked to the user with the properties hasSkill and hasLimitation. A request can also have skills and limitations associated to it, using the properties requiresSkill and excludesByLimitation. A request also has an environment, which can be expressed with the property hasEnvironment.

To keep track of the time-balance of a user, each request is worth a specific amount of time. The property hasAccreditedDuration is used to express this amount of time. The current time-balance of a user is expressed by the property hasTimeBalance.
