Timebank Ontology: Description back to ToC

The two most central concepts are the users and requests. A user can request help and help fellow users. The property isOriginatorOf is used to link a request to a user asking for help. The property accepts is used to link a helper to a request. A user can have skills and limitations, which are linked to the user with the properties hasSkill and hasLimitation. A request can also have skills and limitations associated to it, using the properties requiresSkill and excludesByLimitation. A request also has an environment, which can be expressed with the property hasEnvironment.

To keep track of the time-balance of a user, each request is worth a specific amount of time. The property hasAccreditedDuration is used to express this amount of time. The current time-balance of a user is calculated by adding all time durations for which the user was a helper (acceppted a request) and subtract all time durations for which the user was a receiver of help (originator of a request). The initial time-balance can be expressed by the property hasTimeBalance.

In the following, we proivide some examples on how to use our timebank ontology.

First, we will create a new user, Alice. We will reuse the FOAF vocabulary the indicate the name of the user. The user's address is expressed using the VCard ontology. 


ex:Alice a tb:User ;
    foaf:familyName "Owl" ;
    foaf:firstName "Alice" ;
    tb:hasAddress ex:VCardA .
ex:VCardA a v:VCard .

Next, we link the skill tb:ComputerRepair with Alice and the limitation tb:PoorHearing. 


ex:Alice tb:hasSkill [a tb:ComputerRepair ;
    tb:hasSkillLevel tb:LevelHigh ;
    rdfs:comment "30 years experience with computers." ] .
ex:Alice tb:hasLimitation [a tb:PoorHearing ;
    tb:hasSeverity tb:LevelLow ;
    rdfs:comment "Preferrably, speak into my left ear." ] .

Alice is available every Saturday and Sunday from 1pm until 6pm, starting from 2019-03-30. We use the Time vocabulary to express this. 


ex:Alice tb:isAvailableOn [
    a time:ProperInterval ;
    time:hasBeginning [time:inXSDDateTimeStamp
        "2019-03-30T13:00:00+08:00"^^xsd:dateTimeStamp ] ;
    time:hasEnd [time:inXSDDateTimeStamp
        "2019-03-30T18:00:00+08:00"^^xsd:dateTimeStamp ] ;
    tb:repeatsOn time:Saturday ;
    tb:repeatsOn time:Sunday ] .
Bob wants to create a new request to ask for help to fix his computer. As Bob owns a dog, he excludes users who have a fear of dogs from the request. Since Alice is available, qualified to help, and not afraid of dogs, she accepts the request. 

ex:Request a tb:Request ;
    rdfs:comment "My email client fails to synchronize." ;
    tb:requiresSkill tb:ComputerRepair ;
    tb:excludesUserByLimitation tb:FearOfDogs ;
    tb:hasOriginator ex:Bob ;
    tb:isAcceptedBy ex:Alice ] .
